## OVERVIEW
Chinese Chess (Xiangqi) web application with real-time multiplayer via WebSocket. Express backend + React/Vite frontend, shared game logic.

## STRUCTURE
```
./
├── chess.config.ts       # Runtime config (ports, timeouts, AI settings)
├── package.json          # Root scripts, monorepo-style orchestration
├── backend/
│   ├── src/
│   │   ├── index.ts      # Express entry + HTTP server
│   │   ├── routes/game.ts    # REST API (/api/game/*)
│   │   └── services/
│   │       ├── gameManager.ts    # Game state + rules
│   │       └── gameServer.ts     # WebSocket server (/ws)
│   ├── public/           # Served static assets
│   └── dist/             # Compiled JS output
├── frontend/
│   ├── index.html        # SPA entry
│   ├── src/
│   │   ├── App.tsx       # Root component + game mode logic
│   │   ├── components/   # ChessBoard.tsx (canvas), GameControls.tsx
│   │   └── hooks/        # useGameSocket.ts, useLocalGame.ts
│   └── vite.config.ts    # Build output → backend/public
└── shared/
    ├── types.ts          # All TypeScript types + INITIAL_BOARD
    └── gameLogic.ts      # Pure move validation functions
```

## WHERE TO LOOK
| Task | Location | Notes |
|------|----------|-------|
| Move validation rules | `shared/gameLogic.ts` | `isValidMove()` + 7 piece validators |
| Game state management | `backend/src/services/gameManager.ts` | In-memory Map of games |
| WebSocket protocol | `backend/src/services/gameServer.ts` | MessageType enum + handlers |
| Board rendering | `frontend/src/components/ChessBoard.tsx` | Canvas-based, loads SVG sprites |
| Config runtime | `chess.config.ts` | `loadConfig()` merges env vars |

## CONVENTIONS (this project)
- **Path aliases**: `@shared/*` resolves to `../shared` (frontend only)
- **Backend tsconfig rootDir is `..`** — includes `chess.config.ts` and `shared/` in compilation
- **Frontend builds to `backend/public/`** — not `frontend/dist/`
- **Session auth via express-session** — playerId stored in `req.session`
- **No test infrastructure** — add vitest if needed

## ANTI-PATTERNS (this project)
- **NEVER** commit `node_modules/` — already gitignored
- **NEVER** modify `backend/dist/` — generated by `npm run build:backend`
- **NEVER** hardcode ports — use `chessConfig.server.port`

## UNIQUE STYLES
- **Canvas rendering** for board (not DOM elements)
- **SVG piece sprites** loaded from `/assets/svg/{side}-{type}.svg`
- **Chinese UI labels** in App.tsx ("红方", "黑方", "楚 河", "汉 界")
- **Two game modes**: local (hot-seat) + online (WebSocket)

## COMMANDS
```bash
npm run dev          # Start both backend + frontend (concurrently)
npm run dev:backend  # ts-node-dev backend/src/index.ts
npm run dev:frontend # cd frontend && vite
npm run build        # frontend → backend/public, then tsc backend
npm run start        # node backend/dist/backend/src/index.js
npm run lint         # eslint . --ext .ts,.tsx
npm run typecheck    # tsc --noEmit
```

## NOTES
- Build output `backend/dist/backend/src/index.js` is nested (dist mirrors src structure)
- WebSocket reconnect: 3s delay + 30s ping interval
- Game cleanup: removes finished games after 1hr inactivity
- AI endpoint placeholder exists but `chessConfig.ai.enabled` defaults to false
